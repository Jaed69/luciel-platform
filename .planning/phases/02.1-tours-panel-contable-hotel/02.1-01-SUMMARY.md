**1. [Rule 3 - Blocking] pnpm-workspace.yaml glob doesn't recurse into apps/tours/web/**
- **Found during:** Task 1 (pnpm install)
- **Issue:** Plan assumed `apps/*` resolves `apps/tours` automatically, but pnpm's glob matches `apps/tours/` (no package.json) not `apps/tours/web/`
- **Fix:** Added `'apps/tours/web'` to packages list in pnpm-workspace.yaml
- **Files modified:** pnpm-workspace.yaml
- **Verification:** `pnpm install` now resolves 3 workspace projects (was 2)
- **Committed in:** 7dd0294

**2. [Rule 1 - Bug] Tailwind v4 Vite plugin doesn't work with Next.js 16 (Turbopack)**
- **Found during:** Task 1 (pnpm build — `vite` key not in NextConfig)
- **Issue:** Plan said "Registra @tailwindcss/vite en vite.plugins (mismo patrón que apps/landing/astro.config.mjs)" — but Next.js 16 uses Turbopack, not Vite; the Astro pattern doesn't carry over
- **Fix:** Switched to @tailwindcss/postcss in postcss.config.mjs; removed `vite` key from next.config.ts
- **Files modified:** next.config.ts, postcss.config.mjs, package.json
- **Verification:** `pnpm build` exits 0
- **Committed in:** 7dd0294

**3. [Rule 1 - Bug] Plan's TC interno scenario was mathematically unbalanced + mixed monedas**
- **Found during:** Task 2 (writing test_asientos_admin.py)
- **Issue:** Plan's TC interno example (débito 101-CAJA-PEN 370, crédito 101-CAJA-USD 100, débito 672-GAN-PERD-TC 0) has debe=370 vs haber=100 (unbalanced) AND mixes PEN+USD (violates D-08)
- **Fix:** Changed test to a proper single-moneda PEN asiento (débito 101-CAJA-PEN 100, crédito 672-GAN-PERD-TC 100) with metadata.tipo='tc_interno'; verified audit_log captures metadata
- **Files modified:** apps/tours/api/tests/test_asientos_admin.py
- **Verification:** test_post_asientos_tc_interno_metadata passes (201 + audit_log has metadata.tipo)
- **Committed in:** 2235f6a

**4. [Rule 3 - Blocking] pydantic-settings not in pyproject.toml dependencies**
- **Found during:** Task 2 (pytest — ModuleNotFoundError: pydantic_settings)
- **Issue:** config.py imports `from pydantic_settings import BaseSettings` but pydantic-settings wasn't listed as a dependency
- **Fix:** Added `pydantic-settings>=2.5.0` to pyproject.toml dependencies
- **Files modified:** apps/tours/api/pyproject.toml
- **Verification:** `uv sync` installs pydantic-settings; pytest runs
- **Committed in:** 2235f6a

**5. [Rule 1 - Bug] POST /ventas + POST /asientos session.begin() conflicts with autobegin**
- **Found during:** Task 2 (test_ventas — "A transaction is already begun")
- **Issue:** Route handlers do validation queries (session.execute → autobegins tx) then `async with session.begin()` (tries to begin again → InvalidRequestError)
- **Fix:** Removed explicit `async with session.begin():`; use session autobegin + manual `await session.commit()` / `await session.rollback()`
- **Files modified:** apps/tours/api/app/routers/tours.py, apps/tours/api/app/routers/core.py
- **Verification:** test_ventas + test_asientos_admin pass
- **Committed in:** 2235f6a

**6. [Rule 1 - Bug] audit _snapshot used table.columns (DB names) instead of mapper.column_attrs**
- **Found during:** Task 2 (test_asientos_admin — JSONDecodeError on 'MetaData()')
- **Issue:** `getattr(instance, 'metadata')` returns SQLAlchemy's MetaData object, not the dict value, because `metadata` is a reserved SQLAlchemy attribute; the Python attribute is `metadata_` (aliased via mapped_column)
- **Fix:** Changed _snapshot to use `inspect(instance).mapper.column_attrs` (Python attribute names) instead of `instance.__table__.columns` (DB column names)
- **Verification:** test_post_asientos_tc_interno_metadata passes (audit_log has metadata_ key with tipo='tc_interno')
- **Committed in:** 2235f6a
