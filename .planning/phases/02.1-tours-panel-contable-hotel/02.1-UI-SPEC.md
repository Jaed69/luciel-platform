# Phase 02.1 — UI Design Contract

> Visual and interaction contract for the Tours contable panel — an internal mini-ERP
> (B2B, multi-user, double-entry) replacing the hotel/agencia's Excel+VBA macro sheet.
> Inherits the **Basílica** design DNA verbatim (`.planning/design-systems/basilica.md`) and adapts it
> to a dense accounting surface per the locked "Adaptación para Panel Tours" section of that file.

**Sources pre-populated (do not re-ask):**
- `.planning/design-systems/basilica.md` — full token tables, button/card/nav/input specs, Do's/Don'ts, and the Panel Tours adaptation block at the bottom.
- `02.1-CONTEXT.md` — D-01..D-26 (locked topology, multi-moneda rule, comisión model, audit_log, RBAC).
- `02.1-RESEARCH.md` — stack (Next.js 16 + RSC + Server Actions / FastAPI + SQLAlchemy async + aiosqlite / NextAuth credentials + JWT bridge), project structure, Architecture Responsibility Map.
- `02.1-VALIDATION.md` — verification map drives which screens must exist.
- `.planning/REQUIREMENTS.md` TOURS-01..10, `.planning/ROADMAP.md` SC#1–10.

**Stack lock relevant to UI:**
- Frontend: Next.js 16.2.10 App Router + RSC + Server Actions + TypeScript 5.8 + Tailwind v4.
- UI library: **NO external component library.** Local-only `apps/tours/web/src/components/` primitives built on Tailwind 4. (shadcn = none; Radix/MUI/Chakra = none — out of scope per STATE.md deferred `packages/ui`).
- Server-side data: RSC fetch to `http://tours-api:8000` with `Authorization: Bearer <JWT>` (D-02). No client→API calls except via Server Actions / Route Handlers.
- No PWA, no charts, no Excel/PDF export (deferred to 2.1.x).

## Spacing Scale

Token values verbatim from Basilica (8px base). Do not invent others.

| Token | Value | Usage in Tours panel |
|-------|-------|-------|
| `xxs` | 4px | Icon-to-text gap inside cells, inline padding chips |
| `xs` | 8px | Form input padding `12px 16px` inner (vertical=12, horizontal=16) — vertical 8 used in dense table cells |
| `sm` | 12px | Form input vertical padding, compact chip spacing |
| `md` | 16px | Default element spacing, table cell horizontal padding, dashboard card spacing |
| `lg` | 24px | Card padding (`content-card`), grid gap desktop, form fields vertical rhythm |
| `xl` | 32px | Layout gaps, form section breaks |
| `xxl` | 48px | Major section breaks, dashboard → table break |
| `section` | 80px | Page vertical padding desktop (40–56px mobile per basilica) |

**Exceptions:**
- Touch targets: tab nav items and icon-only buttons `min-h: 44px` (WCAG) even though 44 is not on the 8px scale — the visible pill stays on scale, only the hit-box overflows.
- Accounting table row height: `36–40px` (locked by basilica.md adaptation block — denser than marketing cards).
- Form layout: 2-column desktop grid gap `24px` (`lg`), collapses to 1-column mobile with `16px` (`md`) vertical gap.

**Destructive reserved for (explicit list):**

## Screen-Level Interaction Contracts

The template is a floor, not a ceiling. The following screen contracts give the executor enough specificity to implement without design ambiguity and let the checker validate per-screen interactions.

### S1 — Login (`/login`)

**Layout:**
- Peach-cream full-bleed hero. Centered single column max-w `420px`.
- `arch-niche-divider` at top — amber-glow radial behind arch — used as brand anchor only here (not on authed routes).
- Yeseva One 40px gold "Tours" script accent centered above arch; Playfair 38px "Iniciar sesión" H1 below.

- Submit → Server Action → NextAuth Credentials authorize → FastAPI `/auth/login` → JWT issued.

**RBAC:** unauthenticated route. All authed users redirected away. No nav shown.

### Toastsystem
- Top of viewport fixed bar, height 48px, max-w `640px` centered, `content-card` shadow treatment warm.

### Currency formatting
- Always `Intl.NumberFormat('es-PE', { style: 'currency', currency: 'PEN' })` for S/ and `currency: 'USD'` for $.
- `tabular-nums` class always applied to currency cells.
- Negative → wrap in `<span className="text-chili-red">`.

### Date formatting
- UI render: `format(date, 'dd/MM/yyyy')` (date-fns or native Intl).
- Form input: native `<input type="date">` for accessibility — mask shows DD/MM/YYYY where localized.
- Bridge layer (Server Component / api client) converts ISO ↔ DD/MM/YYYY. Never done in cell renderer.

### Validation
- `floating-label-input` valid state → field bg `wine-light-tint`, label green-ish Wine.
- Invalid → field bg `chili-tint`, label chili-red, helper text Nunito 13 chili-red under field.
- Form-level submit disables `button-primary` until all required valid.

- Next.js `error.tsx` per route group: peach-cream canvas + Playfair 24px "Error del servidor" + Nunito 16 body "Reintenta en unos segundos o contacta al administrador." + `button-outlined` "Reintentar".

- 404: Playfair 24 "Página no encontrada" + Nunito 16 "La URL no existe." + `button-outlined` "Volver al inicio".

## Accessibility Floor (non-negotiable)

- All interactive elements keyboard-reachable; focus-visible ring uses `wine` 2px offset 2px.
- Tables: use real `<table>` with `<thead>/<tbody>`, sticky header is `position: sticky; top: 0`.
- Status badges and icon-only actions → `aria-label` in Spanish (e.g. `aria-label="Eliminar agencia"`, `aria-label="Cerrar liquidación"`).
- Toasts: `role="alert"` for error, `role="status"` for success.
- Tabular numbers for currency always (prevents column jitter).
- Color contrast: Wine #6B1E1C on peach-cream passes AA at heading sizes; chili-red on white passes AA at 14px+ 600. Verify with checker.
