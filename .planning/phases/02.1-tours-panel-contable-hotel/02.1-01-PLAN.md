## Phase Goal

**As a** staff de hotel/agencia, **I want to** loguearme y registrar una venta de tour con un asiento contable balanceado en partda doble, **so that** el sistema reemplace el Excel con macros VBA — sin texto libre, sin ventas sin rastro, con audit log estructural y separación core/módulo Tours.

<objective>
Plan 02.1-01 entrega el primer vertical slice del mini-ERP Tours: scaffold monorepo `apps/tours/{web,api}` + compose Traefik + auth NextAuth/Credentials con bcrypt + JWT bridge a FastAPI + core contable de partida doble con balance validado en Python + audit_log estructural (before_flush + ContextVar) + módulo Tours schema (tours_catalogo, tours_servicios, comision_reglas con default global 50/50) + POST /ventas end-to-end con asiento balanceado + UI: login, layout con nav RBAC, Ventas form con comisión preview live (vía /simular), catálogos CRUD admin (agencias/tours/vendedores/formas-pago/monedas), admin/usuarios CRUD.

Purpose: Establecer el core contable + el inquilino Tours sobre infraestructura reproducible. El Post de Venta produce el asiento balanceado estructuralmente — fulfillment de TOURS-05, el patrón reusado por Plan 02.1-02 para liquidaciones. Sin este plan, Plan 02.1-02 no tiene ledger donde postear los asientos de cierre.

Output: `apps/tours/` con dos servicios corriendo via `docker compose up -d tours-web tours-api`, login → POST venta → asiento cuadrado y auditable. Cierre de liquidaciones + dashboard + deploy HTTPS real se entregan en Plan 02.1-02.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser→tours-web (HTTPS) | Tráfico público del staff over TLS via Traefik. Login es el único unauthenticated endpoint. |
| tours-web SSR→tours-api (HTTP internal, Docker DNS) | Server Components / Server Actions llaman a `http://tours-api:8000` dentro de `traefik-public` network. No sale a internet. |
| tours-api→SQLite WAL (named volume `tours-db-data`) | Single-host, mmap compartido entre procesos del contenedor. |
| NextAuth JWT ⇄ FastAPI pyjwt | Stateless token firmado con `NEXTAUTH_SECRET` compartido. No cookie cross-domain. |
