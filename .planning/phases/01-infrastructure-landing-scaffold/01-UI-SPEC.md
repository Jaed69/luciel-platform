# Phase 1 — UI Design Contract

> Visual and interaction contract for the `apps/landing/` Astro 7 scaffold served at `luciel.dev`. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 scope note:** This is an infrastructure + scaffold phase. The only rendered UI is a minimal placeholder page proving the Traefik/Docker/Let's Encrypt stack serves `https://luciel.dev` with HTTP 200. Real content (home intro, blog, nav, 404, legal pages) is explicitly out of scope — Phase 2/3. This contract defines the token file structure (hook for DESIGN.md) + the placeholder page only.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Astro 7 — shadcn is React-only; gate not applicable) |
| Preset | not applicable |
| Component library | none (scaffold has no components yet; Phase 2 introduces layout/nav) |
| Icon library | none (no icons rendered in Phase 1) |
| Font | Inter (body/UI fallback per DESIGN.md `Unica77 Cohere Web` → `Inter`), Space Grotesk (display fallback per DESIGN.md `CohereText` → `Space Grotesk`), ui-monospace/`SFMono-Regular` (mono fallback per `CohereMono`). No proprietary Cohere fonts bundled — fallbacks only. |
| Styling | Tailwind 4 via `@tailwindcss/vite`, configured with `@theme` block mapping to DESIGN.md tokens. Tokens file is a **structural hook** — values present, not heavily consumed in Phase 1. |

### Token File (Structural Hook)

Per CONTEXT.md canonical_refs: Phase 1 must land a tokens file so Phase 2 content work starts from a configured base, not `npm init`. The `apps/landing/src/styles/tokens.css` (or `@theme` block in a global CSS) captures the full DESIGN.md token set:

- Colors: all 24 swatches from DESIGN.md `colors:` block.
- Typography: all 12 roles from DESIGN.md `typography:` block (font-family stacks use fallbacks only).
- Rounded: 7 radii (xs..full).
- Spacing: 8 DESIGN.md spacing steps.

The Phase 1 placeholder page consumes only the 4-size / 2-weight / 4-color subset declared below. The remaining tokens exist but are unused until Phase 2.

> Registry safety gate: not applicable — no shadcn, no third-party component registry. Nothing to vet.

## Spacing Scale

Declared base scale (multiples of 4 — what the placeholder page actually uses):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (none rendered in Phase 1) |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, placeholder body padding |
| lg | 24px | Section padding, placeholder card padding |
| xl | 32px | Layout gaps, placeholder section breaks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical rhythm |

Exceptions (inherited from DESIGN.md tokens file, NOT used by Phase 1 rendered UI):
- `xxs: 2px` and `xs: 6px` from DESIGN.md spacing — captured in tokens file for Phase 2 consumption; not multiples of 4. Recorded here so the checker sees them acknowledged; Phase 1 placeholder renders zero of these.
- `section: 80px` from DESIGN.md — used by Phase 2 band spacing; declared in tokens file, not rendered in Phase 1.

## Typography

Phase 1 renders 4 roles, 2 weights. Full DESIGN.md 12-role set lives in the tokens file (hook) for Phase 2.

| Role | Size | Weight | Line Height | Font | Usage in Phase 1 |
|------|------|--------|-------------|------|------------------|
| Body | 16px | 400 | 1.5 | Inter | Placeholder paragraph copy |
| Label | 14px | 500 | 1.71 | Inter | Placeholder CTA button label (matches DESIGN.md `button`) |
| Heading | 24px | 400 | 1.3 | Space Grotesk | Placeholder section heading (matches DESIGN.md `feature-heading`) |
| Display | 48px | 400 | 1.2 | Space Grotesk | Placeholder page title (matches DESIGN.md `section-heading`) |

Weights in use: regular (400) + medium (500). No bold/semibold in Phase 1.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (Canvas White) | Page background, placeholder surface |
| Secondary (30%) | `#17171c` (Near-Black Primary) | Footer band, placeholder text blocks on light, dark surface accent |
| Accent (10%) | `#ff7759` (Coral) | Status marker only — see reserved-for list |
| Destructive | `#b30000` (Error Red) | Not used in Phase 1 (no destructive actions). Declared for token-file completeness. |

Accent reserved for: the placeholder status chip ("Under construction" indicator) on the scaffold page. Never the primary CTA (per DESIGN.md, primary CTA is near-black on light surfaces). Coral does NOT appear on any button, link, or broad surface in Phase 1.

> DESIGN.md's broader palette (Soft Stone `#eeece7`, Deep Green `#003c33`, Dark Navy `#071829`, Action Blue `#1863dc`, hairlines, mutes) lives in the tokens file but is not rendered in Phase 1. Phase 2/3 content consumes them.

## Copywriting Contract

Phase 1 renders a single placeholder page at `/`. All copy is English (project language of record per REQUIREMENTS.md / brief).

| Element | Copy |
|---------|------|
| Primary CTA | **"View source on GitHub"** — single text-link CTA on the placeholder, pointing to the project repository. (No user-facing forms or auth in Phase 1; this is the only interactive element.) |
| Page title (Display) | **"luciel.dev"** |
| Page heading (Heading) | **"Under construction"** |
| Empty state heading | **"Nothing here yet"** (the scaffold IS the empty state — content lands Phase 2) |
| Empty state body | **"luciel.dev is being built. Real content — a personal intro, technical blog, and tool directory — lands in the next phase. Follow the build on GitHub."** + next step: link "View source on GitHub" |
| Status chip (accent) | **"Under construction"** (coral chip, mono-label style, uppercase 14px tracked) |
| Error state | *(404 page is Phase 2 scope — CONT-09)* For Phase 1, nginx serves the placeholder page on any unmatched route so the stack never returns a raw nginx 404. If a route truly 404s, copy: **"Page not found. luciel.dev is still being built."** — but full custom 404 is deferred to Phase 2. |
| Destructive confirmation | **None.** Phase 1 has zero destructive user actions. Secret values (`ACME_EMAIL`, `GHCR_PAT`, dashboard basic-auth creds) live in `.env` only — never rendered in UI. |

No forms, no auth, no comments, no interactive widgets in Phase 1. The placeholder is static HTML + minimal CSS — consistent with Astro zero-JS output and AdSense/SEO baseline (CONT-01, SEOA-05 prep).
