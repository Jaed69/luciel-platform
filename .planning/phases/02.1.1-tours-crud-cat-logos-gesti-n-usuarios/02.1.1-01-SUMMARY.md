## Decisions Made
- Per-model `bind_name` + `uses_activo` flags on `_REFERENCED_BY` — lets monedas (no activo column, FK by codigo string) reuse the same code path as integer-PK refs without special cases
- `UsuarioUpdateIn` uses `extra="forbid"` — 422 loud rejection if anyone tries to set the password field via PUT (forces the dedicated /me/password or /{id}/password endpoints)
- Email unique detection via `IntegrityError` + string match on `'usuarios.email' or 'email'` — SQLite-specific but variant-agnostic for the UNIQUE constraint message
